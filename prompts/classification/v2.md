# Prompt: Classification v2

## Objetivo
Clasificar contenido de monitoreo de medios para la marca Claro Colombia.

## Taxonomía de categorías
Asigna exactamente UNA de estas categorías:
- "regulacion" — Regulación, normativa telecom (CRC, MinTIC, ANE)
- "politica_publica" — Política pública, legislación, debates gubernamentales
- "financiero" — Resultados financieros, mercado bursátil, inversiones
- "tecnologia_red" — Infraestructura, 5G, cobertura, conectividad
- "productos_servicios" — Planes, ofertas, tarifas, lanzamientos
- "competencia" — Movimientos de competidores (Movistar, Tigo, WOM, ETB)
- "experiencia_cliente" — Quejas, PQRS, satisfacción, NPS
- "marca_reputacion" — Imagen corporativa, patrocinios, eventos de marca
- "sostenibilidad" — RSE, impacto social, medio ambiente, inclusión digital
- "laboral" — Empleo, sindicatos, cultura organizacional
- "legal" — Demandas, sanciones, litigios
- "seguridad" — Ciberseguridad, fraude, privacidad de datos
- "alianzas" — Partnerships, acuerdos comerciales, fusiones
- "transformacion_digital" — Cloud, IoT, soluciones empresariales
- "otro" — No encaja en ninguna anterior

## Sentimiento
Evalúa el sentimiento HACIA CLARO (no del contenido en general):
- "positivo" — Favorable para Claro: logros, reconocimientos, expansión
- "negativo" — Desfavorable: quejas, sanciones, caídas, problemas
- "neutro" — Informativo sin sesgo, o no menciona a Claro directamente

## Etiquetas
Genera entre 2 y 5 etiquetas descriptivas en snake_case.
Ejemplos: "5g_cobertura", "planes_prepago", "caida_servicio", "ley_tic".

## Resumen
1-2 oraciones (máx 200 caracteres) capturando el punto principal desde la perspectiva de monitoreo de marca Claro.

## Confianza
Valor entre 0.0 y 1.0:
- 0.9-1.0: Claramente sobre Claro con sentimiento inequívoco
- 0.7-0.89: Relevante pero sentimiento ambiguo
- 0.5-0.69: Indirectamente relacionado
- <0.5: Apenas relacionado o datos insuficientes

## Reglas
- Responde SOLO con JSON válido, sin markdown ni texto adicional.
- No inventar datos que no aparezcan en el contenido.
- Si falta información, usar confianza baja y sentimiento "neutro".

## Output esperado
```json
{
  "categoria": "string (de la taxonomía)",
  "sentimiento": "positivo|neutro|negativo",
  "etiquetas": ["string"],
  "confianza": 0.0,
  "resumen": "string"
}
```
